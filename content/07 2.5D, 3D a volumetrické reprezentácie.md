# 07 2.5D, 3D a volumetrické reprezentácie

> *Otázka: 2.5D, 3D a volumetrický 3D obrazom, metódy získavania 3D obrazu, point cloud, voxel grid a polygon mesh.*

## Stručný prehľad

Trojrozmerné dáta môžu byť reprezentované rôznymi spôsobmi, pričom výber závisí od spôsobu ich získania, požadovanej presnosti a následného použitia. *2.5D obraz* je výšková mapa — každému bodu 2D mriežky prislúcha jedna výšková hodnota, bez informácie o previsoch. *Plná 3D reprezentácia* (point cloud, polygon mesh, voxel grid) zachytáva priestor vrátane previsov a zložitých tvarov. *Volumetrický 3D obraz* — špeciálny prípad voxelovej mriežky — uchováva hodnotu atribútu (napr. intenzitu) v každom voxeli a zachytáva aj *vnútorné štruktúry* objektov (CT, MRI). Metódy získavania 3D dát zahŕňajú stereo víziu, Time-of-Flight, štruktúrované svetlo, fotogrametriu, LiDAR a ultrazvuk.

## Rozdiely 2.5D / 3D / volumetric 3D

| Typ | Popis | Príklady |
|---|---|---|
| **2.5D** (*výšková mapa, hĺbková mapa*) | 2D mriežka $(x,y)$, každý bod má jednu hodnotu výšky/hĺbky $z$. Chýba informácia o previsoch — každý $(x,y)$ bod opisuje *len jeden* povrch. | Digitálny model terénu (DEM), hĺbková mapa stereo kamery, výstup ToF senzora |
| **3D** | Plná priestorová reprezentácia $(x,y,z)$; môže popisovať ľubovoľné tvary vrátane previsov, dutín a uzavretých povrchov. Tri hlavné formy: point cloud, voxel grid, polygon mesh. | LiDAR skeny budov, SfM model architektúry, 3D skeny produktov |
| **Volumetrický 3D obraz** (*voxelové pole*) | 3D mriežka voxelov; každý voxel obsahuje hodnotu atribútu (intenzity, hustoty, ...). Zachytáva *vnútorné štruktúry*, nielen povrch. | CT (Hounsfield units), MRI (T1/T2 signál), 3D ultrazvuk |

**Kedy ktorý typ:**
- 2.5D: terén, vozovka, ploche objekty z jednej perspektívy — ľahký na zber a spracovanie
- 3D: komplexné objekty, budovy, sochárske artefakty — treba viacero pohľadov
- Volumetrický: biologické tkanivá, priemyslové súčiastky (kontrola dutín), kryštály

## Metódy získavania 3D obrazu

### 1. Stereo vízia (*Passive Stereovision*)
- Dve kamery z rôznych pozícií → triangulácia zodpovedajúcich bodov
- Výsledok: 2.5D hĺbková mapa (alebo 3D point cloud po transformácii)
- Nevyžaduje aktívne osvetlenie; obmedzenie: textureless plochy
- Podrobne: [[08 Kalibrácia stereo systému a rektifikácia]], [[09 Problém korešpondencie, disparita a hĺbka]]

### 2. Time-of-Flight (*ToF*)
- Vysiela IR impulzy a meria čas odrazu: $d = c \cdot t / 2$
- Produkty: Intel RealSense D400, Microsoft Azure Kinect, Basler ToF kamery
- Výstup: hĺbková mapa + RGB obraz (RGB-D)
- Obmedzenie: problémy s odrazmi, slnečné svetlo, presnosť 1–3 cm

### 3. Štruktúrované svetlo (*Structured Light*)
- Projektor premietne vzor (čiary, náhodné bodky) → kamera sníma deformáciu vzoru → triangulácia
- Microsoft Kinect v1 (IR náhodné body), priemyselné skenovacie systémy (GOM, Keyence)
- Výstup: 3D point cloud s presnosťou až 0,1 mm
- Obmedzenie: vonkajšie prostredie (rušenie slnečným svetlom), lesklé povrchy

### 4. Fotogrametria / SfM
- Rekonštrukcia z mnohých fotografií z rôznych uhlov
- Podrobne: [[10 Structure-from-Motion a Multi-View Stereo]]

### 5. LiDAR (*Laser Imaging Detection And Ranging*)
- Emituje laserové impulzy → meria čas návratu → 3D bod
- **First return**: prvý odraz (vrchol vegetácie, strecha budovy)
- **Last return**: posledný odraz (zem pod vegetáciou)
- Typy: pozemný, letecký (UAV alebo manned aircraft), vozidlový (Velodyne, Ouster, Livox)

**Parametre leteckého LiDARu:**
- *Letová výška*: nižšie → vyššie bodové rozlíšenie (viac bodov/m²), ale menší záber
- *Rýchlosť letu*: vyššia rýchlosť → redšia vzorka pozdĺž trasy
- *Frekvencia snímania (scan frequency)*: napr. 100 kHz impulzov/s → hustejší mrak
- *Uhol skenovania (FOV)*: šírka prúžku pod lietadlom (typicky 20–40°)
- *Presnosť GPS/IMU*: nutná pre georeferencovanie; chyba polohy ~5 cm pri RTK GPS

**Vytvorenie DEM z bodového mraká (LiDAR):**
1. Georeferencovanie: GPS + IMU → absolútne súradnice každého bodu
2. Klasifikácia bodov: „ground" vs. „non-ground" (vegetácia, budovy) — algoritmus Progressive Morphological Filter alebo CSF (Cloth Simulation Filter)
3. Interpolácia: TIN (Triangulated Irregular Network) alebo IDW (Inverse Distance Weighting) → pravidelná rastrová mriežka (napr. 1×1 m alebo 0,5×0,5 m)
4. Výsledok: DEM (Digital Elevation Model) alebo DSM (Digital Surface Model, vrátane vegetácie)

### 6. Ultrazvuk (*USG — Ultrasonography*)
- Vysielač generuje krátke impulzy vysokofrekvenčného zvuku (1–20 MHz)
- Impulzy sa šíria tkanivom; pri prechode medzi materiálmi s rozdielnou *akustickou impedanciou* (napr. sval × kosť) sa časť vlny odrazí
- **Čas návratu** → hĺbka reflektujúcej plochy
- **Amplitúda odrazu** → akustický odpor (hustota × rýchlosť zvuku v tkanive)

**Faktory ovplyvňujúce vracajúci sa pulz:**
- *Rýchlosť zvuku v tkanive*: tukové tkanivo ~1450 m/s, sval ~1540 m/s, kosť ~3500 m/s — rozdiel spôsobuje refrakciu
- *Akustická impedancia* $Z = \rho \cdot c$: čím väčší skok Z, tým silnejší odraz
- *Uhol sondy*: optimálny odraz pri kolmom dopade; šikmý → slabší signál (refrakcia)
- *Absorpcia zvuku*: tkanivá absorbujú energiu (vyššia frekvencia → väčšia absorpcia → menší dosah)
- *Akustický tieň (acoustic shadowing)*: za hustou štruktúrou (kamene) nie je žiaden signál

**3D USG:** mechanický pohyb sondy (lineárny, rotačný) alebo maticové pole senzorov (2D matrix transducer) → séria 2D rezov → volumetrická rekonštrukcia

## Point Cloud, Voxel Grid a Polygon Mesh

### Point Cloud (*Mrak bodov*)

**Definícia:**
Neusporiadaná množina bodov v 3D priestore; každý bod = $(x, y, z)$ + voliteľné atribúty (farba RGB, intenzita odrazu, normála povrchu).

**Vlastnosti:**
- Priamo ukladá výsledok LiDARu, stereo triangulácie, SfM → minimálne spracovanie od senzora
- Flexibilná hustota — môže byť veľmi riedky alebo hustý
- **Žiadna konektivita** — body navzájom nesúvisia (nie je definovaná plocha ani hrana)
- Ľahká editácia: pridanie, odstránenie, filtrácia bodov

**Nevýhody:**
- Renderovanie ťažkopádne bez ďalšieho spracovania
- Hľadanie susedov: lineárne $O(N)$ bez špeciálnych štruktúr; s KD-stromom $O(\log N)$
- Neumožňuje priame volumetrické výpočty

**Formáty:** `.pcd` (Point Cloud Library), `.las`/`.laz` (letecký LiDAR), `.ply`

**Kedy použiť:** surové senzorové dáta (LiDAR, SfM výstup), registrácia a fúzia viacerých skanov, vstup pre segmentáciu (PointNet, PointNet++), geospatial analýza

---

### Voxel Grid (*Voxelová mriežka*)

**Definícia:**
Pravidelná 3D mriežka buniek — *voxelov* (volumetrických pixelov). Každý voxel má diskrétne súradnice $(i, j, k)$ a môže uchovávať rôzne hodnoty (pravdepodobnosť obsadenosti, intenzita CT, príznak povrchu...).

**Vlastnosti:**
- Pravidelná štruktúra → jednoduché indexovanie $O(1)$, priame 3D konvolúcie
- **Kubická pamäťová náročnosť**: objem $N \times N \times N$ voxelov → $O(N^3)$ pamäte. Pre $N=256$: 16 miliónov voxelov — veľký objem, ale zvládnuteľný.
- Pevné rozlíšenie: jemnejšie detaily vyžadujú väčšiu mriežku
- Priama 3D vizualizácia (volume rendering, ray casting)

**Riešenie pamäťovej náročnosti:**
- **Oktostromy (octrees)**: rekurzívne delenie priestoru na 8 podkubusov; prázdne oblasti sa neukladajú → sparse voxel grids
- **OctNet** (Riegler et al. 2017): hybridná reprezentácia pre 3D deep learning

**Formáty:** `.binvox`, TSDF (Truncated Signed Distance Function v TSDF fusion pre real-time 3D rekonštrukciu)

**Kedy použiť:** medicínske 3D zobrazovanie (CT/MRI priamo = voxelová mriežka), 3D-CNN pre rozpoznávanie objektov, occupancy grid v robotickej navigácii, real-time SLAM (KinectFusion, TSDF), herný engine kolízny systém

---

### Polygon Mesh (*Sieť polygónov*)

**Definícia:**
Množina vrcholov $V$ (3D body), hrán $E$ a polygónov (zvyčajne trojuholníkov) $F$, ktoré spoločne definujú uzavretý alebo otvorený povrch. Je to štandardná reprezentácia v 3D grafike.

**Vlastnosti:**
- Efektívne ukladá zložité povrchy s malom počtu polygónov (LOD — Level of Detail)
- **Topológia a konektivita**: každý vrchol vie, ktoré polygóny sú ho susedmi
- **Priame renderovanie** na GPU pomocou OpenGL/Vulkan/WebGL → hry, VFX, AR/VR
- **Texturovanie**: UV mapa premietne fotografiu na mesh → photo-realistic výsledok

**Nevýhody:**
- Komplexná editácia (topológia): topologické operácie (zlúčenie vrcholov, retriangulácia) sú netriviálne
- Ak surové senzorové dáta nemajú konektivitu (LiDAR), treba najprv vytvoriť mesh (Poisson, Ball-Pivoting)

**Formáty:** `.obj`, `.ply`, `.stl` (3D tlač), `.fbx`, `.glb/gltf` (web a AR)

**Konverzia z point cloudu na mesh:**
- *Poisson Surface Reconstruction*: fituje hladký povrch na mrak bodov + normály → smooth, watertight mesh
- *Ball-Pivoting Algorithm (BPA)*: „krúti" guľu po bode → priamo cez body tvorí trojuholníky → bližší pôvodnému tvaru
- *Marching Cubes*: z voxelovej reprezentácie vytvára mesh (izosurface); hladký, ale závisí od rozlíšenia mriežky

**Kedy použiť:** 3D grafika (hry, film, CAD), 3D tlač, digitálna kópia kultúrneho dedičstva, textúrovaný 3D model pre e-commerce, AR/VR

---

**Prehľad — kedy ktorú reprezentáciu:**

| | Point Cloud | Voxel Grid | Polygon Mesh |
|---|---|---|---|
| **Surové senzorové dáta** | ✓ | – | – |
| **Vnútorné štruktúry** | – | ✓ | – |
| **3D grafy / rendeiring** | – | čiastočne | ✓ |
| **Deep learning (3D CNN)** | čiastočne | ✓ | čiastočne |
| **Presný povrch** | – | – | ✓ |
| **Pamäťová efektivita** | ✓ (riedky) | – (kubická) | ✓ (topol.) |

## Matematika

**Hĺbka z ToF:** $d = \frac{c \cdot \Delta t}{2}$

**Triangulácia (stereo):**
$$Z = \frac{f \cdot B}{d}, \quad X = \frac{(u - c_x) \cdot Z}{f_x}, \quad Y = \frac{(v - c_y) \cdot Z}{f_y}$$

**Voxelová indexácia:**
$(x, y, z)_{world} \rightarrow (i, j, k) = \left(\lfloor x/r \rfloor, \lfloor y/r \rfloor, \lfloor z/r \rfloor\right)$, kde $r$ je veľkosť voxela.

## Porovnania a kompromisy

**Presnosť vs. pamäť:**
- Voxel grid 512³ = 134 miliónov voxelov × 1 bajt = 128 MB — bez kompresie neefektívne
- Octree redukuje na ~10× menej pre typické scény s prázdnym vzduchom

**Rekonštrukcia vs. pôvodný tvar:**
- Poisson mesh: hladký, ale môže „vyhladiť" ostré hrany (artefakt)
- BPA: vernejší pôvodnému tvaru, ale môže mať diery

## Časté zlyhania a obmedzenia

- **Lesklé/priesvitné povrchy**: reflexia mätie LiDAR, ToF aj stereo → diery v point cloude
- **Oklúzia**: časti objektu skryté od senzora → chýbajúce body → nutné viacero pohľadov
- **Šum**: každý senzor má šum merania → nutné filtrovanie (Statistical Outlier Removal, Radius Outlier Removal)
- **Fúzia viacerých skanov**: ak sa dva sken posunom nezhodujú, treba registráciu (ICP — Iterative Closest Point)

## Praktické využitie

1. **Autonómne vozidlá** — LiDAR point clouds → object detection (PointPillars, CenterPoint), HD mapy (voxely pre occupancy)
2. **Medicínske zobrazovanie** — CT/MRI voxelové mriežky → 3D segmentácia nádorov (nnU-Net)
3. **Kulturné dedičstvo** — SfM + MVS → point cloud → Poisson mesh → textúrovaný 3D model (Meshy, Meshroom)
4. **Inžinierske projektovanie (BIM)** — TLS (Terrestrial Laser Scanning) → point cloud → mesh → reverzné inžinierstvo
5. **Robotická navigácia** — OctoMap (octree occupancy grid) pre plánovanie pohybu vo 3D
6. **3D tlač a výroba** — STL mesh priamo do tlačiarne; voxelovanie pre FDM planing
